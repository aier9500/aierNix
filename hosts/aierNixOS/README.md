# aierNixOS — Per-Machine Host Entry

This directory is the single source of truth for what runs on the `aierNixOS` machine. It contains exactly three files: the NixOS system entry, the standalone home-manager entry, and the hardware configuration generated by `nixos-generate-config`. No feature logic lives here — this directory only sets values and flips toggles.

---

## Directory Structure

```
hosts/aierNixOS/
├── default.nix              # NixOS system configuration entry point
├── home.nix                 # Standalone home-manager configuration entry point
└── hardware-configuration.nix  # Machine-specific hardware — auto-generated, git-tracked
```

---

## Files

### `default.nix` — NixOS System Entry

Imports every system module (from `modules/system/`) and sets the values that configure this machine. It does three things and nothing else:

1. **Imports** `hardware-configuration.nix` plus all core and feature modules from `modules/system/`.
2. **Sets `myConfig.*`** cross-cutting values consumed by modules:
   ```nix
   myConfig = {
     user     = "aier";
     hostname = "aierNixOS";
     timezone = "America/Caracas";
     locale   = "en_US.UTF-8";
     themeName = "";
   };
   ```
3. **Enables feature toggles** for everything active on this machine:
   ```nix
   mySystem = {
     desktop.gnome.enable    = true;
     desktop.pipewire.enable = true;
     keyd.enable             = true;
     snapper.enable          = true;
     virtualisation.enable   = true;
     flatpak.enable          = true;
     power.enable            = true;
     printing.enable         = true;
   };
   ```

Note that `mySystem.desktop.fonts.enable` is intentionally absent — that toggle exists but is not yet activated (the module body is empty pending a future phase).

Rebuild the system with:
```bash
nh os switch   # alias: sysw
```

### `home.nix` — Standalone Home-Manager Entry

Mirrors the structure of `default.nix` for the user environment. It imports every home module (from `modules/home/`), sets the same `myConfig.*` values, declares the home-manager identity, and enables home feature toggles:

```nix
home = {
  username     = "aier";
  homeDirectory = "/home/aier";
  stateVersion  = "24.05";   # Do not change
};

nixpkgs.config.allowUnfree = true;   # required for vscode, etc.
programs.home-manager.enable = true; # HM manages itself (standalone mode)
```

Feature toggles enabled here:

| Namespace | Toggle | What it activates |
|---|---|---|
| `myHome.shell` | `bash.enable` | PS1, aliases, `y()` wrapper |
| `myHome.cli` | `eza/fzf/zoxide/fastfetch/yazi` | CLI tool suite |
| `myHome.terminal` | `ghostty.enable` | Ghostty terminal emulator |
| `myHome.apps` | `vscode/obsStudio/flatpak/homePkgs` | Applications |
| `myHome.theming` | `gnome/cursors` | dconf fragments + bibata cursors |
| `myHome.fonts` | `enable` | Font symlinks under `~/.local/share/fonts` |
| `myHome.kando` | `enable` | Kando radial menu + autostart |

Rebuild the user environment with:
```bash
nh home switch   # alias: homesw
```

### `hardware-configuration.nix` — Machine Hardware

Auto-generated by `nixos-generate-config`. Documents the physical layout of this machine:

- **CPU:** AMD (kvm-amd kernel module, microcode updates enabled)
- **Kernel modules:** `nvme`, `xhci_pci`, `thunderbolt`, `usbhid`, `sdhci_pci`
- **Filesystem layout:** Btrfs on a single partition with subvolumes for `/`, `/nix`, and `/home`; vfat EFI boot partition
- **Platform:** `x86_64-linux`

**This file must remain git-tracked.** Nix flakes only copy files that are tracked by git. An untracked `hardware-configuration.nix` causes build failures. When setting up a new machine, regenerate with:

```bash
sudo nixos-generate-config --show-hardware-config > hardware-configuration.nix
git add hardware-configuration.nix
```

## Role in the Larger Project

`hosts/aierNixOS/` is an application of the module system defined in `modules/`. The modules contain all logic; this directory contains only declarations of what is true for this specific machine (decision A6 from `DESIGN.md`).

No feature module imports another feature module — composition is this directory's exclusive job.

When a second host is added (e.g., a laptop), it gets its own `hosts/<name>/` directory following the same pattern: a `default.nix`, a `home.nix`, and a freshly generated `hardware-configuration.nix`. The existing modules in `modules/` are reused without modification.

## Dependencies

| Depends on | For |
|---|---|
| `modules/system/*` | All system feature implementations |
| `modules/home/*` | All home feature implementations |
| `modules/options.nix` | `myConfig.*` option declarations |
| `lib/default.nix` | `mkSystem` / `mkHome` wrappers (injected by the flake) |

Nothing else in the repo imports from this directory — the flake's `nixosConfigurations` and `homeConfigurations` outputs reference it directly.
